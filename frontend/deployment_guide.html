网络设备监控系统部署指南

快速搭建您的网络设备监控平台

环境准备

1 系统要求

操作系统: Linux/Windows/macOS

Python版本: 3.7+

Node.js版本: 12+ (仅前端开发需要)

数据库: SQLite (内置)

2 安装依赖

安装必要的Python依赖包:

pip install flask flask-cors paramiko

建议使用虚拟环境:

python -m venv venv
source venv/bin/activate # Linux/macOS
venv\Scripts\activate # Windows

后端部署

3 数据库初始化

系统使用SQLite数据库，首次运行会自动创建以下表结构:

CREATE TABLE IF NOT EXISTS devices (
  ip TEXT PRIMARY KEY,
  status TEXT,
  version TEXT,
  uptime TEXT,
  disk_usage REAL,
  cpu_usage REAL,
  user TEXT,
  group_name TEXT,
  last_check TEXT
);

CREATE TABLE IF NOT EXISTS config (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  auto_refresh_enabled INTEGER,
  refresh_interval INTEGER,
  cmd_version TEXT,
  cmd_uptime TEXT,
  cmd_disk TEXT,
  cmd_cpu TEXT,
  device_groups TEXT
);

4 启动后端服务

运行Flask后端服务:

python backend/app.py

默认启动在 http://localhost:5000

可选配置参数:

# 修改监听端口
app.run(host='0.0.0.0', port=8080)

# 关闭调试模式(生产环境)
app.run(debug=False)

前端部署

5 前端配置

修改API基础URL(如果需要):

// 在monitor.js和config.js中修改
const API_BASE_URL = 'http://your-server-ip:5000/api';

直接打开index.html即可使用，无需编译

6 生产环境部署

建议使用Nginx部署前端:

server {
  listen 80;
  server_name your-domain.com;
  root /path/to/frontend;
  index index.html;
  
  location /api {
    proxy_pass http://localhost:5000;
    proxy_set_header Host $host;
  }
}

测试指引

系统功能测试

1. 添加测试设备

在监控面板点击"添加设备"按钮，输入测试IP(如192.168.1.1)

2. 检查设备状态

等待约10秒后，系统会自动检查设备状态并更新显示

3. 配置SSH连接

在系统配置页面，为设备分组配置正确的SSH凭据

4. 验证监控数据

确认系统能正确显示设备版本、在线时长、资源使用率等信息

常见问题

Q: 设备状态一直显示"离线"

A: 请检查:

  • 设备IP是否正确且网络可达
  • SSH端口是否开放(默认22)
  • SSH用户名和密码是否正确
  • 防火墙是否阻止了连接

Q: 如何修改自动刷新间隔?

A: 在系统配置 → 监控配置中修改"刷新间隔"值(单位:分钟)

Q: 数据库文件在哪里?

A: 数据库文件为项目根目录下的 network_monitor.db
